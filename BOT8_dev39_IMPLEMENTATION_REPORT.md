# BOT8 dev39 实装报告：工资—消费—公司收入回流与分公司初始资金增强版

## 目标

本版根据用户最新决策，驳回 dev38 的劳动裁剪/父代食物线实验方向，转向检查并强化“工资提高 → 个体消费能力/意愿提高 → 市场消费增加 → 货币回流公司”的现金流闭环。

本版仍不加入：人口恢复倍率、最低出生概率、强制出生、强制自给补给、政府凭空创造资源。

## 代码检查结论

dev38 之前：

- 工资提高会提高个体 `balance`，因此会提高订单簿可支付金额上限。
- 但 `individual_buy_willingness` 本身是固定参数，不会因工资收入而提高。
- 因此“消费量随工资提高”部分成立；“消费意愿随工资提高”未显式成立。

## dev39 修改

### 1. 默认工资比例上调

`labor_reward_ratio`：

- 旧值：50
- 新值：60

### 2. 公司/分公司初始资金增强

`company_initial_money`：

- 旧值：1000
- 新值：2000

该总额仍按原有分公司资金比例拆分进入各分公司，增强分公司承受更高工资支出的能力。

### 3. 新增工资响应消费机制

新增人口配置参数：

```python
enable_wage_responsive_consumption = 1
wage_consumption_bonus_per_survival = 10
wage_consumption_bonus_cap = 20
```

机制含义：

- 当回合收到工资的个体，有效买入意愿会随工资收入小幅提高。
- 每获得约 1 回合生存成本规模的工资，增加 `wage_consumption_bonus_per_survival` 点有效买入意愿。
- 最高额外增加 `wage_consumption_bonus_cap` 点。
- 不创造货币，不按人口数量干预，不改变生育概率，只改变有工资个体的消费预算释放速度。

### 4. 新增诊断字段

- `EffectiveBuyWillingnessAvg`
- `WageConsumptionBonusAvg`
- `WageResponsiveBuyerCount`
- `WageResponsiveExtraCapTotal`
- `WageFundedMarketSpending`
- `WorkerMarketSpending`
- `WorkerMarketSpendingToCompany`

## 结论

1. 工资提高方向有效，尤其显著改善 5 人长期测试：从 0/10 存活提升到 3/10 存活。
2. 10 人×1000 提升到 10/10 存活，现金流仍守恒。
3. 工资提高确实带来更高个体消费与更高公司收入回流。
4. 公司初始资金 2000 比 3000 更适合当前模型；3000 在快速增长后更容易放大资源压力，5人长期表现反而更差。
5. 仍未实现 5 人长期稳定。当前剩余问题是：高工资增强了人口增长与消费，但也同步放大食物、医疗、生育用品压力。

## 下一步建议

建议 dev40 继续沿工资—消费—公司收入闭环方向细化，而不是回到人口恢复倍率或强制干预：

1. 细测 `labor_reward_ratio = 58, 60, 62` 与 `company_initial_money = 1800, 2000, 2200` 的组合。
2. 增加“工资支出后公司现金保护”诊断：不是阻止发工资，而是观察哪些分公司在高工资后现金低于生产资源购买线。
3. 检查高工资下新增死亡主要来自 `food_shortage` 还是 `critical_goods_shortage`，决定是调食物分公司资金比例还是医疗分公司资金比例。
4. 若 5 人测试仍在后期灭绝，优先调整分公司初始资金分配比例，而不是继续单纯增加总资金。
